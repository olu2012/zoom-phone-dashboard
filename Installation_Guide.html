1. Prerequisites

System Requirements

Component Requirement
Operating System Windows 10/11 or Windows Server 2016+
RAM Minimum 4GB (8GB recommended)
Disk Space 500MB minimum
Node.js Version 18.x or higher (LTS recommended)
Network Internet access for Zoom API calls

Required Software

  • Node.js - Required to run the backend server
  • IIS - Optional, for production deployment of frontend
  • Web Browser - Chrome, Edge, or Firefox (latest version)

2. Installing Node.js

1Download Node.js

Visit the official Node.js website: https://nodejs.org

Download the LTS (Long Term Support) version for Windows.

2Run the Installer

  • Double-click the downloaded .msi file
  • Click "Next" to proceed through the installation wizard
  • Accept the license agreement
  • Use the default installation path (or choose your preferred location)
  • Ensure "Add to PATH" is checked
  • Click "Install" and wait for completion

3Verify Installation

Open Command Prompt (cmd) and run:

node --version
npm --version

You should see version numbers displayed (e.g., v18.17.0 and 9.6.7).

Success: If both commands display version numbers, Node.js is installed correctly.

3. Extracting Application Files

1Extract the ZIP file

Extract the provided zoom-phone-dashboard.zip to your desired location, for example:

C:\Applications\zoom-phone-dashboard

4. Backend Setup

1Navigate to the backend folder

Open Command Prompt and navigate to the backend directory:

cd C:\Applications\zoom-phone-dashboard\backend

2Install dependencies

npm install

This will install all required Node.js packages (express, axios, cors, dotenv).

3Configure Environment Variables

Open the .env file in the backend folder and configure your Zoom API credentials:

# Zoom API Credentials (obtain from Zoom Marketplace)
ZOOM_ACCOUNT_ID=your_account_id_here
ZOOM_CLIENT_ID=your_client_id_here
ZOOM_CLIENT_SECRET=your_client_secret_here

# Server Configuration
PORT=3000

# Environment
NODE_ENV=production
Important: Replace the placeholder values with your actual Zoom API credentials. These can be obtained from the Zoom App Marketplace by creating a Server-to-Server OAuth app.

Obtaining Zoom API Credentials

  1. Go to https://marketplace.zoom.us
  2. Sign in with your Zoom account
  3. Click "Develop" > "Build App"
  4. Choose "Server-to-Server OAuth"
  5. Fill in the required information
  6. Copy the Account ID, Client ID, and Client Secret
  7. Add the required scopes: phone:read:admin, phone:read:call_log:admin

5. Frontend Setup

1Navigate to the frontend folder

cd C:\Applications\zoom-phone-dashboard\frontend

2Install dependencies

npm install

This may take a few minutes as it downloads Angular and all dependencies.

3Configure API URL (if needed)

If your backend runs on a different server or port, update the environment file:

Edit src/environments/environment.prod.ts:

export const environment = {
  production: true,
  apiUrl: 'http://YOUR_SERVER_IP:3000/api'
};

4Build for Production

npm run build

This creates a dist/zoom-phone-dashboard folder with the compiled application.

6. IIS Configuration (Production)

Note: IIS is optional. For simple deployments, you can run both frontend and backend using Node.js. IIS is recommended for production environments where you need better performance and management.

Option A: Using IIS (Recommended for Production)

1Enable IIS on Windows

  • Open Control Panel > Programs > Turn Windows features on or off
  • Check "Internet Information Services"
  • Expand and also check:
    • Web Management Tools > IIS Management Console
    • World Wide Web Services > Common HTTP Features (all)
    • World Wide Web Services > Application Development Features > .NET Extensibility, ASP.NET
  • Click OK and wait for installation

2Install URL Rewrite Module

Download and install from: https://www.iis.net/downloads/microsoft/url-rewrite

3Create IIS Website

  1. Open IIS Manager (search "IIS" in Start menu)
  2. Right-click "Sites" > "Add Website"
  3. Configure:
    • Site name: ZoomPhoneDashboard
    • Physical path: C:\Applications\zoom-phone-dashboard\frontend\dist\zoom-phone-dashboard
    • Port: 80 (or your preferred port)
  4. Click OK

4Add URL Rewrite Rule

Create a web.config file in the dist folder:

<?xml version="1.0" encoding="UTF-8"?>
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="Angular Routes" stopProcessing="true">
          <match url=".*" />
          <conditions logicalGrouping="MatchAll">
            <add input="{REQUEST_FILENAME}" matchType="IsFile" negate="true" />
            <add input="{REQUEST_FILENAME}" matchType="IsDirectory" negate="true" />
          </conditions>
          <action type="Rewrite" url="/index.html" />
        </rule>
      </rules>
    </rewrite>
    <staticContent>
      <mimeMap fileExtension=".json" mimeType="application/json" />
    </staticContent>
  </system.webServer>
</configuration>

Option B: Without IIS (Development/Simple Setup)

You can serve the frontend using a simple Node.js static server or use Angular's development server:

cd C:\Applications\zoom-phone-dashboard\frontend
npm start

This will serve the frontend at http://localhost:4200

7. Running the Application

Starting the Backend Server

Open Command Prompt and run:

cd C:\Applications\zoom-phone-dashboard\backend
npm start

You should see: Server running on port 3000

Tip: To run the backend as a Windows service (auto-start on boot), consider using tools like PM2 or NSSM.

Installing PM2 for Production (Recommended)

npm install -g pm2
cd C:\Applications\zoom-phone-dashboard\backend
pm2 start server.js --name "zoom-dashboard-api"
pm2 save
pm2 startup

Starting the Frontend

If using IIS: The frontend is automatically served. Access via http://localhost or your configured port.

If not using IIS:

cd C:\Applications\zoom-phone-dashboard\frontend
npm start

Accessing the Dashboard

Setup Type Frontend URL Backend URL
Development http://localhost:4200 http://localhost:3000
IIS Production http://localhost (or configured port) http://localhost:3000

8. Troubleshooting

Common Issues

Problem Solution
"node" is not recognized Restart Command Prompt after Node.js installation, or reinstall Node.js ensuring "Add to PATH" is checked
npm install fails Run Command Prompt as Administrator, check internet connection
Backend won't start Check if port 3000 is already in use. Change PORT in .env file if needed
Cannot connect to API Verify backend is running, check firewall settings, verify API URL in environment.ts
Zoom API errors Verify API credentials in .env file, ensure OAuth app has correct scopes
IIS 404 errors Install URL Rewrite module, add web.config file
CORS errors in browser Ensure backend is running and CORS is configured correctly

Checking Logs

Backend logs are displayed in the Command Prompt window where the server is running.

For PM2 logs:

pm2 logs zoom-dashboard-api

Firewall Configuration

If accessing from other computers on the network, ensure these ports are open:

  • Port 3000 (Backend API)
  • Port 80 or 4200 (Frontend, depending on setup)
